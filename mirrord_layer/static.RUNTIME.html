Main tokio Runtime for mirrord-layer async tasks.

Apart from some pre-initialization steps, mirrord-layer mostly runs inside this runtime with RUNTIME.block_on.

Usage

Currently it’s being used to run 2 big tasks:

  1. [connection::connect]: which creates the mirrord-agent connection for this layer instance;

  2. start_layer_thread: where we spawn mirrord-layer’s main loop.

Bypass

To prevent us from intercepting neccessary (local) syscalls (like creating a socket), we use detour::detour_bypass_on on_thread_start, and detour::detour_bypass_off on_thread_stop.
