Expand description

Handles the differences between hyper’s HTTP/1 and HTTP/2 connections.

Thanks to this trait being implemented for both v1::HttpV1 and v2::HttpV2, we can have most of the implementation for dealing with requests in ConnectionTask.

Mainly used for our retry connection mechanism, where we create a new HttpV implementor, but we want to swap the “failed attempt” HttpV::Sender with the newly created one.
